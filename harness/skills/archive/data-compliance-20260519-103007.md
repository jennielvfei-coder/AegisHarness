# Calibrated Prediction (Prophet)

## When to Use
Use this skill whenever a user requests a **numerical probability estimate** about a future event, especially under uncertainty, or when they want a structured forecast with explicit reasoning that can be scored later for accuracy (Brier score). Triggers include:
- "What's the probability that X happens by Y date?"
- "Give me a calibrated forecast on…"
- "Predict whether…"
- Requests for multiple linked predictions about a domain trend

## How To

### 1. Frame the Question
Restate the user's question as a precise, falsifiable prediction with a clear **resolution date** and **observable criterion**. If ambiguous, ask for clarification before proceeding.

### 2. Build the Causal DAG
- Identify key factors that influence the outcome.
- Draw a Directed Acyclic Graph (conceptually — describe nodes and edges in text).
- **Check for factor independence** — note where factors are correlated and adjust to avoid double-counting evidence.

### 3. Set Dual Priors
Establish two independent baselines before seeing specific evidence:
- **History-based prior**: What is the base rate of similar events in comparable domains? (e.g., "in the last 10 years, X% of analogous technologies reached mainstream adoption within 18 months of a major demo")
- **Mechanism-based prior**: What does a first-principles model of the causal forces suggest, ignoring historical base rates? (e.g., "given the incentives, friction, and enabling conditions, what probability emerges from a simple model?")

If the two priors diverge significantly, pause and diagnose why — this often reveals hidden assumptions.

### 4. Apply Bayesian Updating (Log-Odds)
Start from the more reliable prior (usually history-based), then update using a **likelihood-ratio tier system**:
| Tier | Evidence Strength | LR Range |
|------|-------------------|----------|
| Weak | Suggestive but noisy signal | 1.5:1 – 3:1 |
| Moderate | Clear, relevant signal | 3:1 – 10:1 |
| Strong | Highly diagnostic signal | 10:1 – 30:1 |
| Decisive | Near-deterministic signal | 30:1+ |

For each piece of evidence, assign a tier, convert to log-odds, add to the prior, and convert back to probability. Document each step explicitly so the reasoning is auditable.

### 5. Pre-Mortem & Bias Self-Check
Before finalizing, perform these counterfactual exercises:
- **Pre-mortem**: Imagine it is now the resolution date and the prediction was wrong. Write down the most plausible reason *why* it was wrong. Does this reveal an overlooked factor?
- **Bias checklist**: Check for overconfidence, recency bias, narrative fallacy, and availability heuristic. Adjust the final probability if any bias is detected.

### 6. Apply Contextual Minimums
If the prediction touches on domains with structural uncertainty floors (e.g., geopolitical events, breakthrough technology timelines), ensure the final probability respects a reasonable minimum (rarely below 5% or above 95% for one-year-out events).

### 7. Output the Report
Present predictions in this structure:
```
Prediction: [falsifiable statement]
Resolution Date: [date]
Final Calibrated Probability: [P%]
Confidence Interval: [range]
Key Factors: [bulleted list]
Pre-Mortem: [most likely failure mode]
Bias Adjustments: [what was corrected and why]
```

For multiple predictions, include a summary table at the top.

### 8. Offer Persistence (Brier Score Tracking)
After delivering predictions, **always offer** to save them as a markdown file to the project workspace. The file should include the resolution date so future sessions can compute Brier scores. If the user accepts, write to a path like `predictions/<topic>-<date>.md`.
