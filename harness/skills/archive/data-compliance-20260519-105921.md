# 多源新闻抓取与分析工作流

## When to Use

- 用户触发"今日新闻"或类似每日新闻聚合指令
- 需要从**中文源**（人民网/财新/财联社/36氪）和**英文源**（Google News/arXiv）并行抓取新闻
- WebFetch 返回 `"Unable to verify if domain is safe to fetch"` 错误时
- 目标：生成结构化日报（8类总览 + 重点分析 + 因果追踪 + 预测信号）

## How To

### 第0步：WebFetch 可用性诊断

WebFetch 在执行前向 `api.claude.ai` 发送安全预检请求。若该域名在当前网络环境不可达（如中国大陆），**所有** WebFetch 调用均会失败，与目标网站连通性无关。

**快速诊断**：
```
WebFetch 任意域名 → 若返回 "Unable to verify if domain is safe to fetch" → 预检失败
```

### 第1步：修复 WebFetch（首选方案）

在 `settings.local.json` 中添加：
```json
{
  "skipWebFetchPreflight": true
}
```

验证修复：
```
WebFetch people.com.cn → 应返回正常HTML内容
WebFetch arxiv.org → 应返回论文列表
```

**注意**：此设置绕过安全预检，仅应在信任目标源时使用。始终向用户说明风险。

### 第2步：降级方案（预检修复不可用时）

按优先级排列的三层降级：

| 优先级 | 方案 | 覆盖范围 | 适用场景 |
|--------|------|----------|----------|
| 1 | **World News API** | 全球聚合 + 人民日报链接 | 快速获取结构化新闻摘要 |
| 2 | **browser-use MCP** | 任意网站（真实浏览器渲染） | 需要完整页面内容时 |
| 3 | **extract_news_links** | 人民网（200+链接已验证） | 中文源深度抓取 |

**并行策略**（推荐）：
```
同时触发：
  - World News API: get_top_news(us,en) + get_top_news(cn,zh)
  - World News API: search("AI model") + search("NVIDIA chip") + search("geopolitics") + search("AI regulation")
  - WebFetch: arxiv.org（学术论文）
  - WebFetch: people.com.cn（中文要闻）
```

### 第3步：结构化日报生成

输出固定模板：

```

## 1. 八类总览表
| 类别 | 新闻条数 | 关键主题 |
|------|----------|----------|
| AI产业 | N | ... |
| 芯片/算力 | N | ... |
| 地缘政治 | N | ... |
| 宏观经济 | N | ... |
| 政策/监管 | N | ... |
| 科技/学术 | N | ... |
| 社会/文化 | N | ... |
| 其他 | N | ... |

## 2. 重点分析（3-5篇）
每篇包含：事件摘要 + 用户关联度 + 行动启示

## 3. 因果追踪
- 已验证假设
- 新假设
- 矛盾/冲突对

## 4. Prophet 预测信号
- 5条可操作信号，含验证窗口

## 5. 数据源状态
- 各数据源可用性
- WebFetch 修复记录
```
