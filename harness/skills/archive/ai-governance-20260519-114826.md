# China WebFetch Fix

## When to Use
- When WebFetch calls to Chinese websites (e.g., people.com.cn, ce.cn, cls.cn, 36kr.com) or even global sites (news.google.com, arxiv.org) all fail with: **"Unable to verify if domain is safe to fetch"** while operating from a mainland China network.
- During any automated scraping or news workflow that depends on `web_fetch`, and the preflight check to `api.claude.ai` is blocked.

## How To
1. **Confirm the root cause**: All `web_fetch` attempts fail with the same preflight error, even for domains that are otherwise reachable. The preflight sends a request to `api.claude.ai` (blocked in China) before fetching.
2. **Apply the fix**: Add `"skipWebFetchPreflight": true` to the active settings file (e.g., `.claude/settings.local.json` in the workspace, or global `~/.claude/settings.json`).
   ```json
   {
     "skipWebFetchPreflight": true
   }
   ```
3. **Retry**: Retry the same WebFetch calls. Chinese sources (people.com.cn, 36kr.com) and international ones (arxiv.org, news.google.com) should now succeed.
4. **Fallback strategies if the fix doesn't cover all needs**:
   - **World News API**: Use `search_news` or `get_top_news` for aggregated Chinese/global news; `extract_news_links` can pull headlines even when body extraction is truncated.
   - **browser-use MCP**: Launch a real browser to navigate to JavaScript-heavy or CAPTCHA-protected sites (e.g., 36kr.com newsflashes) and extract rendered content.
5. **Document the fix** in your workflow file (e.g., `daily-news-trigger.md`) so future sessions can instantly re-apply it if the preflight restriction returns after tool updates.
